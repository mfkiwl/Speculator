Binary file format

Binary files are used for two purposes:

  • To store mixture covariance matrices of some multichannel audio signals (Rx.bin)
  • To store the average energy per frequency of some multichannel audio signals (Rx_en.bin)

For each purpose, one format is adopted and described below.

Rx.bin

We need to store one complex square matrix per frequency bin \(F\) per time frame \(N\). The dimension of each matrix is the number of audio channels \(I\). Given the fact that each matrix is Hermitian ( \(M_{i,i}\in \mathbb{R}\) and \(M_{i,j} = \overline{M_{j,i}}\)), we designed the file format so that we store each matrix as a real vector of size \(I \times I\).

Header

In the header, the data is stored as integers, which size is equal to 4 bytes on most computers.

The first integer is the number of dimensions, which is always 3 in our case ( \(I \times I\), \(F\) and \(N\)).

Then we store each dimension as an integer.

Data

The data is stored as floats, which size is also equal to 4 bytes.

For each time frame and for each frequency bin, we store \(I \times I\) floats. The \(I\) first floats are the real diagonal elements.

Then we store the upper triangular part of the matrix. The real and imaginary parts of the complex elements are always stored side by side: the real part of \(M_{i,j}\) is always followed by the imaginary part. These elements are stored in the following order: we store the first row ( \(M_{1,2}\), \(M_{1,3}\) until \(M_{1,I}\)) then the second row ( \(M_{2,3}\), \(M_{2,4}\) until \(M_{2,I}\)) and so on until the last element \(M_{I-1,I}\).

Rx_en.bin

We need to store one double value per frequency bin \(F\).

Header

In the header, the data is stored as integers, which size is equal to 4 bytes on most computers.

The first integer is the number of dimensions, which is always 1 in our case ( \(F\)).

Data

Data are stored as double values, which size also equal to 8 bytes.

For each frequency bin, we store one double value.
