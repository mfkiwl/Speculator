Calculates the pitch of a set of data.

====================================================

Uses the FLWT described in Eric Larson and Ross Maddox's paper. The algorithm was optimized for embedded systems that do not have dedicated hardware for floating point arithmetic by removing any floating point operation that was not deemed necessary to calculate the pitch. This of course decreases the accuracy of the algorithm, but at the same time increases the efficiency of the algorithm. If a more precise calculation of pitch is desired, check out the URL below for a floating point implementation of this algorithm.

Parameters
dataPointer to array of data
datalenLength of the array
fsSampling frequency of data
Returns
Pitch of the window
Return values
pitch
  • 0.0 : Determines the data is pitchless
  • > 0.0 : Found a pitch
See also

http://www.schmittmachine.com/dywapitchtrack.html

Calculates the pitch of a set of data.

====================================================

Uses the same algorithm as described in FLWT::getpitch. If the new pitch is either twice or half the frequency of the previous window, the new pitch is considered an error and the return value is the pitch of the last window.

Calculates the pitch of a set of data using a median filter.

====================================================

Uses a median filter to calculate the median on the past 5 frequencies including the newest one calculated from the new set of data. Uses the same algorithm as described in FLWT::getpitch. This has a slightly larger overhead to return the pitch.
